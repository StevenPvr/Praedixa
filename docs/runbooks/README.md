# Runbooks

Ce dossier contient les procédures opératoires pour exploiter, diagnostiquer et sécuriser Praedixa.

## À quoi sert un runbook ici

- Décrire des séquences d'actions reproductibles.
- Réduire le temps de réaction en incident ou en release.
- Éviter les opérations "tribales" connues d'une seule personne.

## Exemples présents

- gates locales et sécurité ;
- rotation de secrets ;
- readiness go-live ;
- vérification d'intégrité supply chain ;
- orchestrateur medallion ;
- réponse rapide à un zero-day.

## Public visé

- développeurs fullstack ;
- owner technique ;
- personnes en charge du déploiement ou de la sécurité.
