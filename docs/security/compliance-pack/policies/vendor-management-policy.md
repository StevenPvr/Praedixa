# Vendor Management Policy

## Purpose

Define onboarding and review of vendors/subprocessors.

## Requirements

- Risk classification before onboarding.
- DPA and security terms reviewed.
- Data residency documented at country level.
- Annual or quarterly review based on risk.
- Exit plan documented for critical vendors.

## Residency gate (mandatory)

A vendor is not approved for production customer data unless:

1. France data residency is contractually stated.
2. Technical configuration proves storage and processing in France.
3. Backup and log locations are documented as France.

## Minimum controls

- Named owner per vendor.
- Evidence of latest review.
- Open issues tracked with due dates.
- No production go-live without residency gate sign-off.
