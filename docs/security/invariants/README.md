# Invariants de Sécurité

Ce dossier contient les invariants de sécurité exprimés sous forme de fichiers YAML.

## Fichiers

- `security-invariants.yaml` : contrôles attendus côté plateforme.
- `owasp-control-coverage.yaml` : cartographie vers les contrôles OWASP.
- `abuse-scenarios.yaml` : scénarios d'abus à prévenir ou détecter.

## À quoi ils servent

- Aligner la sécurité documentaire, les audits et les scripts de gate.
- Rendre explicites les hypothèses de défense à préserver quand le code évolue.
