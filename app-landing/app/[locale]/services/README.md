# Services

## Rôle

Ce dossier matérialise un segment de route Next.js. Les fichiers `page.tsx` définissent le rendu, le layout ou le handler HTTP de ce segment.

## Contenu immédiat

Sous-dossiers :

- Aucun élément versionné direct.

Fichiers :

- `page.tsx`

## Intégration

Ce dossier est consommé par l'application `app-landing` et s'insère dans son flux runtime, build ou test.

## SEO

Cette route fait partie des pages pivots pour la recherche de marque: elle doit garder une breadcrumb visible et un balisage `WebPage` + `BreadcrumbList` alignes avec le titre public de la page.

## Positionnement

La route `services` porte l'offre publique canonique: `un premier perimetre de decision`, pas une page de choix entre plusieurs produits.
Elle doit rester concrete sur `ce qui est livre`, `ce qui n'est pas livre`, `ce qu'il faut cote client`, `qui participe` et le rythme des `30 jours`, avec la preuve sur historique seulement comme mecanisme d'entree si le premier arbitrage doit etre objectivise avant lancement.
